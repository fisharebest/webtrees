DESCRIPTION

back to top
A basic new sidebar element for each Individual’s page showing the current privacy status of that individual (living, dead, or "presumed dead"), with an explanation of why that is the status.

It is designed to assist in identifying the cause of an individual being marked as dead (and therefore displayed pubicly on many sites), when they are actually expected to be living and therefore hidden.

For further information about this, please read this forum discussion: http://www.webtrees.net/index.php/en/forum/2-open-discussion/27740-privacy-issues and this WIKI article: http://wiki.webtrees.net/Access_and_Privacy.

REQUIREMENTS

back to top
This module requires webtrees version 1.4.1 or later.

INSTALLATION

back to top
  1. Unzip the files and upload them to the webtrees /modules_vX/ folder on your web server. (Note: "X" = "3" at the time of writing this, but may change in future releases)
  2. Login to webtrees as administrator, go to AdministrationModules, and find the module. It will be called "Privacy status". Check it has a tick for "Enabled"
  3. Click SAVE, to complete the installation.
  4. OPTIONAL. You can also go to AdministrationModules/Sidebar and change the position of this tab relative to all others. Simply drag and drop the Privacy status sidebar up or down the list as required, then click SAVE.
  5. OPTIONAL. In the same place you can set the access level to determine who can see this information. I would recommend a setting of "show to managers" or "show to editors". Then click SAVE.

UPGRADING

TRANSLATION

back to top
To translate the unique texts in this module requires the use a "custom translation" file. Full details of how to create and use one is on the webtrees WIKI at http://wiki.webtrees.net/Custom_Translation

If anyone does translate the texts please forward a copy of the translation to me, then it can be included in future versions of the module.
