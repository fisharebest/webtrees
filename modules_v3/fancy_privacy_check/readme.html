Description

back to top
A basic new sidebar element for each Individual’s page showing the current privacy status of that individual (living, dead, or "presumed dead"), with an explanation of why that is the status.

It is designed to assist in identifying the cause of an individual being marked as dead (and therefore displayed pubicly on many sites), when they are actually expected to be living and therefore hidden.

For further information about this, please read this WIKI article: http://wiki.webtrees.net/Access_and_Privacy.

Requirements

back to top
This module requires webtrees version 1.5.0 or later.

Installing the module

back to top
Installation is very simple. Upload the folder 'fancy_privacy_check' to your server's webtrees /modules_v3 folder. Then login to webtrees as administrator and go to Module Administration (Admin -> Module Administration). Look down the list of installed modules for "Fancy Privacy Check" and ensure it is ticked as "enabled".

After the module is installed go to administration/modules/sidebar to change the position of this tab relative to the other sidebar tabs. Here you also can set the access level. I would recommend a setting of 'show to editors' or higher.

Updating from a previous version

back to top

In case of updating you can just overwrite your previous version of the Fancy Privacy Check module with the new one by uploading the folder fancy_privacy_check' to your server's webtrees /modules_v3 folder. All settings will be remained after an update. It is always a good behaviour to clear the cache after each update. Go to your webtrees administration - clean up data folder and tick the box 'cache'.

Custom translations

back to top

Where possible the Fancy Privacy Check module uses existing webtrees language items. These will therefore translate automatically wherever they are used. We have included new language items in the same "WT_I18N::translate()" format. This means you can translate these into any other language using webtrees "custom translation" feature, described here: Wiki: Custom translation. In stead of adding your custom translation file(s) to the folder data/language, you may opt to keep them seperate from other translated files by adding them to the folder modules_v3/fancy_treeview/language. Csv of php files as decribed on the Wiki could be used as well.

You can use the default.pot file added in the module language folder, to start translate the module in other language(s). The default.pot file contains all strings needed to be translated. You can open this file in any texteditor. Save your file as *.po file. To use the translated *.po file on the module you need to convert this file to a *.mo file. This can be done with a programm like Poedit. But it is not mandatory. Just send me the translated *.po file and I will turn it in to a *.mo file and add it to the module package.

The Dutch translation file is already added to the module language folder. If you have made your own translation or updated an existing one, please send me a copy.
