Description

back to top
This is a very small module with a very powerful purpose. It provides the ability to include the excellent "MySQLDumper" backup software into your webtrees administration panel options. Throughout webtrees documentation you are very sensibly advised to back up your database at frequent intervals, and especially before any upgrade. If your webhost provides a tool like PhpMyAdmin or cPanel you should, if you know how, be able to use that. But for greater flexibility, simplicty and convenience MySQLDumper is hard to beat.

Requirements

back to top

This module requires webtrees version 1.5.0 or later.

You also need a working installation of MySQLDumper. This is NOT provided as part of this module. See Installation below.

Installing the module

back to top

Before doing anything with these files, download and install the latest version of MySQLDumper from www.mysqldumper.net. To use it in conjunction with this module directly, place the folder from the downloaded zip in the root of your webtrees installation. Rename the folder to mysqldumper. If you prefer another location and/or name for this tool please read the additional instructions below.

After installation, access MySQLDumper directly by going to your_webtrees_installation/mysqldumper to set it up and to check it is fully functional. There is no point wasting time with this module until you have a working installation of MySQLDumper. NOTE: If you have any problems installing or using MySQLDumper please contact them at their website for support. They provide excellent service

After you setup MySQLDumper, upload the folder 'fancy_database_backup' to your server's webtrees/modules_v3 folder. Then login to webtrees as administrator and go to Module Administration (Admin -> Module Administration). Look down the list of installed modules for "Fancy Database Backup" and ensure it is ticked as "enabled".

IMPORTANT: In order to retrieve the admin panel for this module you must refresh your browser ("F5" in most browsers).

Additional Setup

back to top

If you have installed the mySQLDumper folder and files in a different location of your webtrees installation, you have to point the module to this other location.

Open the file modules_v3/fancy_database_backup/module.php. At line 54 replace the entry "mysqldumper/index.php" with the correct path to the index.php file of your own MySQLDumper installation. Then save the revised file back to your web server.

Updating from a previous version

back to top

In case of updating you can just overwrite your previous version of the Fancy Database Backup module with the new one by uploading the folder 'fancy_database_backup' to your server's webtrees/modules_v3 folder. All settings will be remained after an update. It is always a good behaviour to clear the cache after each update. Go to your webtrees administration - clean up data folder and tick the box 'cache'.

Do not make any changes to your separate MySqlDumper folder or files. Just take care if you modified line 54 as described in ADDITIONAL SETUP above. It will need the same change in the new file.

Custom translations

back to top

Only in the admin area the module uses a few language strings that are not in the webtrees translations files already. If you are the only administrator of your website and you can understand the English strings, there is no need to translate these. But for those who wants to translate these strings you can copy the file modules_v3/fancy_database_backup/language/en-US.php in the same folder and rename it to your language code used by webtrees (leave the extension as is). To find the right language code look in the folder webtrees/language. Change the second part of each string (the part after the '=>') to strings in your language and save the file. You can use any texteditor for this purpose.

Clear de cache from your admin panel in order to see the changes immediately.
